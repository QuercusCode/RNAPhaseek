## What was fixed this session (10 of 23 confirmed findings resolved)

### Architecture / model code
- ✅ `species_registry.py`: 25/25 heuristic tests pass; HIVEP1/2/3, RSVP1
  no longer misclassified as virus; Ensembl ID prefixes (`ENST`, `ENSG`,
  `ENSMUST`, `ENSDART`, `ENSRNOT`, `ENSXETG`) + MOD DB accessions
  (`MGI:`, `RGD:`, `WBGene`, `FBgn`, `TAIR`) handled via regex with
  `\d{5,}` suffix; `_VIRUS_RE` uses lookbehind/lookahead `[^a-z]` so
  underscore-delimited names (`Rotavirus_A`, `Influenza_A_virus_RNA`)
  match correctly.
- ✅ `RNAPhaseek_hybrid_config.py`: `n_species` now `_REGISTRY_N_SPECIES`
  (single source of truth).
- ✅ `RNAPhaseek_hybrid.py`: `__init__` asserts `args.n_species ≥ N_SPECIES`;
  `forward()` defaults `species_id` to `SPECIES_TO_ID["unknown"]` when
  `use_species_embed=True` but `species_id=None` (preserves head input
  shape on the legacy inference path).
- ✅ `configure_optimizers`: `species_emb.weight` routes to weight_decay=0
  group.

### Data pipeline
- ✅ `unified_merge.py`: detection-first `ensure_species_in_header`,
  module-level `MAX_N_FRAC` / `MAX_SINGLE_BASE_FRAC` constants,
  `passes_quality()` helper, SHA1 `_canon_hash`, intra-write dedup,
  label-leak exclusion against positive hash set, `_resolve_cdhit_bin()`

## What remains — 13 confirmed findings still pending

### HIGH severity (block multi-species training)

1. **Dataset / collate / trainer wiring for `species_id`** —
   `HybridRNADataset.__getitem__` doesn't yield `species_id`,
   `make_collate_fn` doesn't batch it, `evaluate()` and the train loop
   don't unpack it or pass to `model(...)`. Mitigated by the new
   default-fallback in `forward()` (won't crash) but you'll get
   constant `unknown` IDs unless wired through.
   _Files:_ `Functions/RNAPhaseek/RNAPhaseek_hybrid_data.py:99-106,124-165`,
   `Functions/RNAPhaseek/RNAPhaseek_hybrid_trainer.py:48-64,229-240`.

2. **FEGS cache is index-keyed**; reusing the Phase-1
   `fegs_topk_pos/`/`fegs_topk_neg/` for the multispecies pool silently
   mis-pairs sequences. **CRITICAL silent bug.** Need to regenerate the
   cache against the new `unified_all_{positives,negatives}.fasta`,
   write a SHA1 sidecar in each `.npz`, and verify boundary hashes at
   load time.
   _Files:_ `Functions/precompute_fegs.py`,
   `Functions/RNAPhaseek/RNAPhaseek_hybrid_train.py:87-106`.

3. **Stratified species × label split needed**. With this skew
   (yeast/drosophila/virus underrepresented), a stratified-by-label-only
   split could land entire (species, label) cells into one fold.

### MEDIUM severity (degraded training, silent confusion)

4. `generator_hybrid.py` doesn't accept a `--species` flag; SeqProp
   design currently uses `unknown` for all generated sequences.
5. Biophysical features include 9 human-tuned RBP motifs with no
   species awareness; per-species mean/std logging would help debug.
6. `smOOPs_mouse` header convention currently double-prefixes to
   `llps_mouse|smOOPs_mouse_…|` — cosmetic but mismatches the docs.
7. Loss balancing per species (yeast 218 vs mouse 2,822 = 13× skew;
   consider per-species weights or focal loss).

### LOW severity (cleanup)

8. Operator-precedence cleanup on the smoops/mouse rule has already
   been applied; the dead-code block on old lines 71-80 removed.
9. Yeast salvage `Q[0-9]+` mitochondrial entries — backup contains
   them; the new curated yeast file replaces them anyway.
10. Viral pool has zero matched negatives (current entry in
    `prepare_multispecies_negatives.py` intentionally skips virus).
    Generate dinucleotide-preserving shuffles if class balance matters.
11. Duplicate FASTA headers `RNAPS0000288`, `RNAPS0000295` still in
    upstream `all_positives_dedup.fasta` (different sequences, same
    RPS ID — segmented Influenza genome). Cosmetic.
12. Trace species (`zebrafish`/`xenopus`/`arabidopsis` < 5) are
    side-effects of the better registry. Either (a) treat as "trace"
    and exclude from per-species evaluation/stratification (keep in
    pool for diversity), or (b) targeted ingestion later.
13. `n_species` magic constant duplication — RESOLVED (now imported).
    Keeping the listing for completeness.
